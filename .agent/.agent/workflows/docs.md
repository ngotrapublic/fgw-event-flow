# Documentation Management Workflow

This workflow invokes the **Docs Manager Agent** to create and maintain documentation.

## When to Use

- When creating README files
- When documenting APIs
- When writing technical guides
- When updating existing docs

## Instructions

1. **Read the docs manager agent instructions**:
   - Read the file `.gemini/agents/docs-manager.md` for detailed guidance

2. **Assess documentation needs**:
   - Identify what needs to be documented
   - Check existing documentation
   - Determine target audience

3. **Write documentation**:
   - Use clear, concise language
   - Include code examples
   - Add diagrams where helpful
   - Structure content logically

4. **Review and validate**:
   - Verify accuracy of information
   - Test code examples
   - Check for broken links
   - Ensure consistency

5. **Organize documentation**:
   - Use consistent file naming
   - Create logical folder structure
   - Add table of contents

## Documentation Types

| Type | Purpose | Location |
|------|---------|----------|
| README | Project overview | `/README.md` |
| API Docs | API reference | `/docs/api/` |
| Guides | How-to tutorials | `/docs/guides/` |
| Architecture | System design | `/docs/architecture/` |

## Writing Guidelines

- **Be concise**: Short sentences, clear explanations
- **Be specific**: Include concrete examples
- **Be consistent**: Follow style guide
- **Be helpful**: Anticipate reader questions

## Output Format

Provide documentation report with:
- Documents created/updated
- Sections covered
- Outstanding gaps
- Recommendations
