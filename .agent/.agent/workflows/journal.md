# Journal Writing Workflow

This workflow invokes the **Journal Writer Agent** to document progress and learnings.

## When to Use

- When documenting daily progress
- When recording learnings
- When reflecting on decisions
- When capturing insights

## Instructions

1. **Read the journal writer agent instructions**:
   - Read the file `.gemini/agents/journal-writer.md` for detailed guidance

2. **Gather content**:
   - Review what was accomplished
   - Identify key learnings
   - Note challenges faced

3. **Write the entry**:
   - Follow consistent format
   - Be specific about details
   - Include code snippets if relevant
   - Reflect on decisions made

4. **Organize and store**:
   - Use consistent naming convention
   - Store in appropriate location
   - Tag for easy retrieval

## Journal Format

```markdown
# Journal Entry - [Date]

## Summary
Brief overview of the day/session.

## Accomplishments
- Task completed 1
- Task completed 2

## Learnings
- What I learned about X
- Insight about Y

## Challenges
- Problem faced and how resolved

## Next Steps
- What to do next
```

## Output Format

Create journal entry in:
- docs/journals/YYMMDD-topic.md
