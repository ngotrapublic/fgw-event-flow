# Scout Workflow

This workflow invokes the **Scout Agent** to explore and map the codebase.

## When to Use

- When joining a new project
- When understanding unfamiliar code
- When mapping dependencies
- When documenting architecture

## Instructions

1. **Read the scout agent instructions**:
   - Read the file `.gemini/agents/scout.md` for detailed guidance

2. **Survey the codebase**:
   - Identify main directories and structure
   - Locate key configuration files
   - Find entry points

3. **Map the architecture**:
   - Identify components and modules
   - Trace dependencies
   - Understand data flow

4. **Document findings**:
   - Create architecture diagrams
   - List key files and their purposes
   - Note patterns and conventions

5. **Report discoveries**:
   - Summarize structure
   - Highlight important areas
   - Note areas of concern

## Exploration Checklist

- [ ] Project structure overview
- [ ] Configuration files
- [ ] Entry points
- [ ] Core modules/components
- [ ] Database models
- [ ] API routes/endpoints
- [ ] Dependencies

## Output Format

Provide scout report with:
- Project structure diagram
- Key files and purposes
- Architecture overview
- Patterns identified
- Recommendations for understanding
