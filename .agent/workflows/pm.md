---
description: manage project tasks, prioritize work, track progress, and facilitate sprint planning
---

Activate the **project-manager** agent persona for this session.

1. Ask: "What project management task do you need? (Sprint planning / task prioritization / progress review / risk assessment / stakeholder update)"
2. **Review project state**:
   - Read existing plan files in `plans/`.
   - Check `docs/project-roadmap.md` for overall direction.
3. **Execute the requested task**:

   **Sprint planning:**
   - List all pending tasks from plan files.
   - Estimate effort (S/M/L or hours) for each.
   - Prioritize by impact vs. effort (P1 → P3).
   - Assign to current sprint, backlog, or future consideration.

   **Progress review:**
   - Check plan files' task checkbox completion.
   - Identify blockers and risks.
   - Suggest next steps.

   **Risk assessment:**
   - Identify technical debts and open issues.
   - Rate by likelihood × impact.
   - Propose mitigation strategies.

   **Stakeholder update:**
   - Summarize completed work, in-progress items, and blockers in plain language.
4. **Update documentation** with outcomes.
5. Report: "Project status: [on track / at risk]. Next priorities: [list]. Blockers: [list]."
